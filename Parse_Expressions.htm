Parse Expressions

Variations of the Parse operator are used to extract specific information from log messages. These include:

  • Parse Anchor
  • Parse Regex or Extract
  • Parse Using Path/to/Parser Library
  • Parse nodrop Option

Parse Anchor is used to extract a string based on start and stop anchor points, and then to alias the extracted string as a user-created field. Parse Regex, or Extract, uses regular expressions to extract more complex or nested information as aliased fields. The "Parse Using..." operator applies a predefined library of rules to create a series of named fields from standard log formats. For more information, see Automatically Parsing Fields with the Apache Parser.

The "nodrop" option is appended to any parse query to include messages in results that do not meet the matching restrictions of the parse operator. For example, if your search query uses a parser library like Cisco-ASA, then messages that do not match any of the parser rules are normally dropped. If you include the nodrop option, messages that do not match any of the parser rules are included with results.
