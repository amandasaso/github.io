Parsing multiple values in a field

In addition to parsing a field, the multi option allows you to parse multiple values within a single field. This means that Multi keyword instructs the parse regex operator to not just look for the first value in a field but for all of the values, even in messages with a varying number of values in a field. As a part of this process, the multi keyword creates copies of each message so that each individual value in a field can be counted.

For example, say our firewall log messages look like this:

http://www.bigdata.biz" "CU1_4919|967:925:123

From this message, we'd like to extract the firewall codes. We'll use the multi keyword in our parse regex:

... parse url first
| parse regex "Firewall Rules: \|(?<trigger_rules>.*?)\|"
| parse regex field=trigger_rules "(?<trigger_rule>\d+)" multi
