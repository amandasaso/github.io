Data Forwarding

This feature is only available to customers with Enterprise accounts, or those in paid trials. If you'd like to try Data Forwarding, please contact your sales representative.

When enabled, Data Forwarding feature allows Sumo Logic to upload data to an S3 bucket that belongs to your organization. Log messages are returned right after being ingested by Sumo Logic.

Important: Sumo Logic can’t monitor the amount of data being uploaded to the S3 bucket. Please keep an eye on the size of the bucket, and follow any internal policies for managing data returned to your organization.

To set up Data Forwarding:

Note: Data can only be sent to a single S3 bucket.

  1. Make sure to have your AWS Key ID and Secret Key set up to allow Sumo Logic to write to the S3 bucket. You can learn more here.
  2. In the Sumo Logic Web Application, choose Manage > Data Management.
  3. Click Edit.



  4. Enter the exact name of the S3 bucket, along with the AWS Key ID and Secret Key generated for Sumo Logic. Make sure that Active is checked. Then click Save.



After the credentials are verified by AWS data will begin uploading to the S3 bucket.

 

Editing Data Forwarding Settings

If you’d like to temporarily stop forwarding data, you can inactivate the S3 bucket. Additionally you can delete the existing bucket to permanently stop data forwarding, or to add a new bucket.

To cancel data forwarding:

  1. In the Sumo Logic Web Application choose Manage > Data Management.
  2. Click Edit.
  3. Click Remove.
  4. Click Save.

To delete S3 information:

Important: Once the Remove button is clicked there is no way to undo this operation; the S3 information is immediately deleted.

  1. In the Sumo Logic Web Application choose Manage > Data Management.
  2. Click Edit.
  3. Click Remove.
