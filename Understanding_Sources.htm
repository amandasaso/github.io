Understanding Sumo Logic Sources

A Source is a logical partition for the data that a Collector gathers and sends to the Sumo Logic Cloud. You can set up as many Sources as you need for a given Collector. A Source should be configured to collect similar data types. For example, you might set up one Remote File Source to collect printer and router activity logs, and another Remote Source to collect status messages from appliances.

Each Source is tagged with its own metadata. The more Sources you set up, the easier it is to isolate one of the Sources in a Search since each Source can be identified by its metadata. For each installed Collector, you can configure any number of Sources of these four types: Local File, Remote File, Syslog, and Windows Event.

Local File

A Local File Source actively collects from files on the same machine where the Collector is installed. Collecting with a Local File Source can provide an easy solution for some complex configurations. For example, you can set up a Syslog server to output messages to a local file and then collect the local file.

Remote File

A Remote File Source actively collects logs and data from files in remote machines (machines without a Collector installed) using an SSH connection. When collecting remote files from Windows machines, you have two choices: collect via SSH using a third-party client, or set up a Local File Source (not a Remote File Source) to collect using CIFS/SMB by entering a UNC Share path when you configure the Source. See detailed instructions on setting up your Windows environment to collect using CIFS.

Syslog

This Source acts as a passive Syslog server that receives logs and data from multiple devices using the syslog protocol (over TCP or UDP). If you are already running a syslog server, you can either switch to using a Sumo Logic Collector as your syslog server (recommended) or you can output the existing syslog server data to a local file, and then set up a Local or a Remote File Source to collect the file.

Windows Event Log

This Source type collects Windows Event Logs from local Windows computers, or from remote Windows machines using native WMI (Windows Management Instrumentation). Before collecting remote Windows Event Logs, see detailed set up instructions for your Windows environment.

Script

If you need to collect data that isn't stored in log files (such as database records, or system performance metrics) you can use a Script source to fetch custom sources of data. The script executes at defined intervals.

You can add new Sources to a Collector or edit the configuration of your current Sources at any time.
